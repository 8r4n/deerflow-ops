# Index issues (pinned) and how to maintain them

Pinned index issues act as the table-of-contents for this repo’s memory.

Create these 3 issues and pin them in the GitHub UI:

## 1) Skill Registry Index
Purpose: links to every `memory:skill` issue.

Structure:
- New skills (last 30 days)
- Stable skills
- Deprecated skills
- “How to add a skill” (link to relevant playbook)

Maintenance rule:
- When a new `memory:skill` issue is created, add it here in the same run.

## 2) Repo Evaluation Index
Purpose: links to key `memory:repo` evaluations (especially adopted/rejected decisions).

Structure:
- Adopted
- Rejected
- Revisit later

Maintenance rule:
- Any time a repo evaluation is created/updated with a decision, reflect it here.

## 3) Run Journal Index
Purpose: quick navigation across `run:log` issues.

Structure:
- Current week
- Prior weeks/months

Maintenance rule:
- When a `run:log` is created, add it here immediately.
- When a run finishes, mark it (done/blocked) in the index.

## Anti-noise rule
Verbose details live in `run:log` issues. Only curated, reusable content should be promoted into `memory:*` issues.
